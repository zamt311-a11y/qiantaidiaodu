# 账号管理 + 统一登录 + CSV 编码兼容 设计说明

**目标**
- 增强账号管理：超级管理员/管理员/工程师三级权限，支持新增/编辑/重置密码/删除（停用）
- 新增单一登录页 `/login`，按角色自动跳转
- CSV 导入支持自动识别编码，仍乱码时提供手动编码选择

**范围**
- 后端：角色权限校验、用户管理接口、CSV 编码解析
- 前端：统一登录页、管理员账号管理页、导入页编码选择 UI、页面访问自动跳转

---

## 1. 权限模型与访问规则

- 角色：`super_admin` / `admin` / `engineer`
- 权限规则：
  - `super_admin`：可管理所有账号；可新增管理员与工程师
  - `admin`：仅管理工程师账号
  - `engineer`：无账号管理权限
- 页面访问：
  - 未登录访问任何页面 → 跳转 `/login`
  - 登录后按角色自动跳转：
    - `super_admin/admin` → `/admin/tasks`
    - `engineer` → `/m/home`

## 2. 账号管理页面（管理员端）

- 新增“账号管理”页面：
  - 表格字段：姓名、手机号、角色、状态、创建时间、操作
  - 操作：编辑、重置密码、停用/启用、删除（软删除）
  - 新增/编辑抽屉统一表单
  - 角色下拉限制：
    - `super_admin` 可选管理员/工程师
    - `admin` 仅可选工程师
  - 重置密码：后端生成随机密码并返回一次

## 3. CSV 编码兼容（导入/预览）

- 后端新增编码检测函数：
  - 默认自动识别顺序：`utf-8-sig` → `utf-8` → `gb18030` → `gbk` → `gb2312` → `utf-16`
  - 若手动选择编码则优先使用
- 前端导入页新增编码选择下拉：
  - 选项：自动识别、UTF-8、UTF-8 BOM、GB18030、GBK、GB2312、UTF-16LE、UTF-16BE
  - 自动识别失败或乱码时，允许用户切换编码重新解析

## 4. 技术改动点

- 后端
  - 更新 `require_admin` 允许 `super_admin`
  - 新增 `require_super_admin`（如需硬性限制）
  - 用户接口权限细化（创建/编辑/删除/重置）
  - CSV 解析函数统一支持编码参数
- 前端
  - 新增 `login.html` + Logo
  - 所有页面增加“未登录 → 跳转 /login”的统一校验
  - 管理端导航新增“账号管理”
  - 导入页新增编码选择与重新解析

---

**成功标准**
- 超级管理员/管理员/工程师权限生效
- 登录后按角色跳转正确
- CSV 导入不再因编码导致乱码，必要时可手动修复
