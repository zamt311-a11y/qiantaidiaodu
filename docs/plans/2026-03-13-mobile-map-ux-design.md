# 移动端地图体验优化 设计说明

**目标**
- 移动端地图自动显示 GPS 定位红点（类似高德更新频率）
- 任务点更易点击、筛选结果点击后自动关闭并定位
- 任务操作面板增加“提交/取消”
- 路线图层可开关（规划路线 + 我的路线）
- 任务页地图入口更醒目

---

## 1. 定位与标记交互

- 页面加载即尝试定位：`navigator.geolocation.watchPosition`
- 以红点表示当前位置，定期更新位置（高精度 + 10s 内最大缓存）
- 定位失败时提示但不阻塞地图加载

## 2. 任务点点击与筛选结果

- 移动端任务点扩大点击热区：采用 HTML Marker 外层包裹
- 筛选结果列表点击任务后：
  - 自动关闭筛选结果弹窗
  - 定位到任务点并弹出信息窗

## 3. 任务操作面板

- 操作按钮变为“选择状态” → “提交/取消”
- 提交前可修改异常类型/备注
- 取消回退到未选择状态

## 4. 路线图层控制

- 新增“显示规划路线 / 显示我的路线”开关
- 关闭仅隐藏线路与节点，不清空路线数据
- 再次开启自动恢复显示

## 5. 管理端任务页地图入口强化

- 在任务页顶部增加醒目的“打开地图”按钮
- 保留原导航菜单，提升可发现性

---

**成功标准**
- 移动端地图自动显示红点定位
- 任务点更易点击，筛选结果点击后自动关闭并定位
- 任务状态操作含提交/取消
- 路线图层可随时开关
- 任务页地图入口显著清晰
