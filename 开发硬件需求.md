# 10人规模网优工具极致简化方案（无对象存储/无Redis，成本最低）
针对“省去对象存储+微型Redis缓存”的需求，核心逻辑是**用“应用服务器本地存储”完全替代外部服务**——10人规模数据量极小（工参文件≤100MB/年、现场照片≤500MB/年、数据库备份≤100MB/月），服务器本地硬盘完全能承载，且省去所有额外服务成本，全程仅需1台应用服务器，部署更简单。


## 一、核心简化点：用服务器本地存储替代2项服务（无额外成本）
### 1. 替代对象存储：服务器本地建“文件目录”存所有文件
不再采购对象存储（OSS），直接在应用服务器的硬盘上新建3个目录，分别存储对应文件，10人用量完全够用：  
- **工参文件目录**：存导入的4G/5G工参Excel/CSV（单文件≤20MB，1年≤50个，总容量≤1GB）；  
- **现场照片目录**：存工程师上传的异常现场照片（单张≤5MB，10人日均≤5张，1年≤1.8GB）；  
- **备份文件目录**：存数据库每日备份（单备份≤100MB，保留30天，总容量≤3GB）。  
- **安全保障**：通过服务器“文件权限设置”（开发层实现，无成本），仅允许后端服务读写这些目录，禁止外部直接访问，和对象存储安全级别一致。

### 2. 替代Redis缓存：用“后端本地缓存”替代（性能足够）
不再采购Redis缓存，直接在后端代码中用“内存缓存”（如Java的`HashMap`、Python的`dict`）缓存高频数据：  
- 缓存内容：管理员/工程师账号权限、常用工参图层配置、当天任务列表（数据量≤10MB）；  
- 适用场景：10人日均查询≤100次，本地缓存响应速度（毫秒级）完全满足需求，无需额外Redis；  
- 优势：无部署成本，无需配置Redis连接，代码更简单。

## 三、极致简化后部署流程（仅需1台服务器，1天内完成）
全程只部署“1台应用服务器”，所有服务（后端+数据库+文件存储+缓存）都在这台机器上，步骤减少50%：

### 1. 服务器初始化（1小时）
1. 登录阿里云ECS控制台，找到已购服务器：  
   - 配置防火墙：仅开放2个必需端口（80端口：网页端访问；443端口：HTTPS加密），**关闭所有其他端口**（包括数据库3306端口，只允许服务器本地访问）；  
   - 改服务器密码：用“字母+数字+特殊符号”（8位以上），开启“登录IP限制”（仅允许公司/个人常用IP登录，更安全）。

### 2. 1台服务器装完所有服务（4小时，全免费）
用远程工具（FinalShell，免费）连接服务器，按顺序安装4个组件（都开源免费，有一键安装脚本）：  
| 组件               | 用途                  | 安装步骤（傻瓜式）                                                                 |
|--------------------|-----------------------|-----------------------------------------------------------------------------------|
| JDK 1.8            | 后端代码运行环境      | 执行命令：`yum install java-1.8.0-openjdk`（CentOS系统自带，10分钟完成）           |
| 本地MySQL 8.0      | 存储账户/任务/工参数据 | 执行命令：`yum install mysql-server`，安装后设置密码，新建“netopt_tool”数据库（20分钟完成） |
| Tomcat 8.5         | 跑后端+网页端代码     | 下载Tomcat压缩包，解压到`/usr/local/`目录，启动服务（10分钟完成）                  |
| 本地文件目录       | 存工参/照片/备份      | 执行命令：`mkdir -p /netopt/file/工参 /netopt/file/照片 /netopt/file/备份`，设置目录权限（10分钟完成） |

### 3. 部署代码+初始化数据（2小时）
1. 上传代码：  
   - 后端代码（.jar/.war文件）上传到Tomcat的`webapps`目录；  
   - 网页端代码（HTML/CSS/JS）上传到Tomcat的`webapps/ROOT`目录；  
2. 初始化数据：  
   - 用Navicat（免费版）连服务器本地MySQL，导入“表结构”（含用户表、任务表、工参表）；  
   - 手动添加1个管理员账号（密码用BCrypt加密后存入，避免明文）；  
3. 测试后端接口：访问“http://服务器IP:8080/api/test”，显示“数据库连接成功+文件目录可读写”即完成。

### 4. 部署SSL证书+消息推送（1小时）
1.  SSL证书：下载阿里云免费DV证书，部署到Tomcat（修改`server.xml`配置文件，10分钟完成），访问“https://服务器IP”显示“小绿锁”；  
2.  消息推送：在APP代码中集成极光免费SDK，测试推送功能（30分钟完成，无需额外服务）。

### 5. 配置本地备份脚本（30分钟，防数据丢失）
在服务器上写1个简单的`shell`脚本（复制即用，无需懂代码），每天凌晨自动备份数据库到本地“备份目录”：  
```bash
# 数据库备份脚本（示例）
mysqldump -u 数据库账号 -p'数据库密码' netopt_tool > /netopt/file/备份/$(date +%Y%m%d)_backup.sql
# 保留最近30天备份，删除更早的
find /netopt/file/备份 -name "*.sql" -mtime +30 -delete
```
- 执行方式：通过`crontab`设置定时任务（每天凌晨2点执行），1次配置永久生效；  
- 双重保险：每月手动下载1次备份文件到本地电脑（如U盘），防止服务器硬盘损坏。


## 四、关键注意事项（保障稳定，无额外成本）
### 1. 服务器硬盘容量别省（100GB足够，成本差极小）
- 必须选100GB系统盘（比80GB仅多100-200元/年），预留足够空间存文件+备份，避免后续硬盘满了无法使用；  
- 定期查看硬盘使用情况（通过服务器控制台“磁盘监控”，免费），当使用率超过70%时，手动删除1年前的旧工参/照片（10人1年数据量小，删除后不影响）。

### 2. 数据安全靠2个免费操作
- 文件权限：设置本地文件目录仅允许“后端服务用户”（如`tomcat`用户）读写，其他用户无权限（防止误删）；  
- 密码加密：数据库存密码用BCrypt加密，后端缓存不存明文密码，HTTPS传输加密账户密码（全程无明文）。

### 3. 后续扩展无压力
- 若后续用户增加到20人，仅需升级服务器配置（如2核4G→4核8G，年成本增加300-500元），无需重新部署服务；  
- 若需要恢复对象存储/Redis，直接在现有服务器上新增配置即可，无需更换服务器。
